git/github As someone who knew nothing at all of github, it can as somewhat of a surprise to learn that github sprung up out of providing a central bank of git repositories (projects). These repositroies being either shared privately or publicly, enabling a secure development facility based on the git process. The git process being a versioning system. I registered with github several years ago in order to get components and ideas for java projects but I never used it for my own versioning and security. I tended to manually backup my work at suitable restore points. Now that I have started using git, I can see the advantages of git to enforce work disciplines. I needed to get my mind a round the concept of repositories, branches, commits, pushes and pulls.
